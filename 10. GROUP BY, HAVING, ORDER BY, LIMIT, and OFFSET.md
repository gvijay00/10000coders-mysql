## ğŸ”¹ 2. HAVING Clause

###  Definition:

`HAVING` is used to **filter group-level results** (i.e., after `GROUP BY`). It's like a `WHERE` clause for groups.

###  Syntax:

```sql
SELECT column1, AGGREGATE_FUNCTION(column2)
FROM table_name
GROUP BY column1
HAVING AGGREGATE_FUNCTION(column2) condition;
```

###  Example:

```sql

FROM employees
GROUP BY department
HAVING COUNT(*) > 5;
```

ğŸ”¹ *This shows only departments that have more than 5 employees.*

> ğŸ”¸ **Note:** `WHERE` filters **before** grouping, `HAVING` filters **after** grouping.

## ğŸ”¹ 3. ORDER BY Clause

###  Definition:

`ORDER BY` is used to **sort the result set** by one or more columns.

###  Syntax:

```sql

FROM table_name

```

###  Example:

```sql

FROM employees

```

ğŸ”¹ *This sorts employees by salary from highest to lowest.*

> ğŸ”¸ Default sorting is `ASC` (ascending). Use `DESC` for descending.

## ğŸ”¹ 5. OFFSET Clause

###  Definition:

`OFFSET` skips the first N rows before starting to return the rows.

###  Syntax:

```sql

LIMIT number OFFSET offset_value;
```

OR

```sql

LIMIT offset_value, number;
```

###  Example:

```sql

LIMIT 5 OFFSET 10;
```

ğŸ”¹ *Skips the first 10 rows and shows the next 5.*

> ğŸ”¸ Equivalent to:

```sql

LIMIT 10, 5;
```
